## 2. 목적

개인 혹은 소규모 팀이 작업 항목을 시각적으로 관리할 수 있는 칸반보드 웹 애플리케이션을 제공한다. 사용자는 카드를 생성·이동·삭제하며 TO-DO → In-Progress → Done 흐름으로 작업 상태를 추적할 수 있다. v2에서는 Supabase 인증 및 DB 연동을 통해 다중 기기에서 동일한 보드를 사용할 수 있다.

---

## 3. 목표 사용자

- 개인 태스크를 체계적으로 관리하고 싶은 개발자 및 기획자
- 별도의 설치 없이 브라우저만으로 사용할 수 있는 경량 도구가 필요한 사람
- 여러 기기에서 동일한 칸반보드에 접근하고 싶은 사용자

---

## 4. 핵심 기능 (v2)

| # | 기능 | 설명 |
|---|------|------|
| F-01 | 컬럼 구성 | TO-DO, In-Progress, Done 3개 컬럼 고정 표시 |
| F-02 | 카드 추가 | 각 컬럼 하단 입력창에 텍스트 입력 후 Enter 또는 버튼 클릭으로 카드 생성 |
| F-03 | 카드 삭제 | 카드의 × 버튼 클릭으로 카드 제거 |
| F-04 | 드래그 앤 드롭 | 카드를 다른 컬럼으로 드래그하여 상태 이동 (HTML5 Drag and Drop API) |
| F-05 | 상태 영속성 | Supabase DB에 저장, 다중 기기에서 동기화 |
| F-06 | 이메일 인증 | 이메일/비밀번호 회원가입 및 로그인 |
| F-07 | Google 소셜 로그인 | Google 계정으로 OAuth 로그인 |
| F-08 | GitHub 소셜 로그인 | GitHub 계정으로 OAuth 로그인 |
| F-09 | 로그아웃 | 세션 종료 및 로그인 화면 복귀 |

## 6. 사용 시나리오

### 시나리오 1 — 회원가입 및 첫 로그인
> 신규 사용자가 이메일로 계정을 만들고 보드에 접근한다.

1. 브라우저에서 앱을 연다 → 로그인 화면이 표시된다.
2. 이메일과 비밀번호를 입력하고 "회원가입" 버튼을 클릭한다.
3. **결과**: 확인 이메일이 발송되고, 링크 클릭 후 로그인하면 빈 보드가 표시된다.

---

### 시나리오 2 — 소셜 로그인
> Google 계정으로 빠르게 로그인한다.

1. 로그인 화면에서 "Google로 계속하기" 버튼을 클릭한다.
2. Google 인증 화면에서 계정을 선택한다.
3. **결과**: 앱으로 리다이렉트되고 보드가 표시된다.

---

### 시나리오 3 — 새 작업 등록
> 로그인 후 TO-DO 컬럼에 카드를 추가한다.

1. TO-DO 컬럼 하단 입력창에 "디자인 시안 검토"를 입력한다.
2. Enter 키를 누른다.
3. **결과**: TO-DO 컬럼에 카드가 생성되고 Supabase DB에 저장된다.

---

### 시나리오 4 — 작업 진행 상태 변경
> 카드를 In-Progress 컬럼으로 이동한다.

1. TO-DO 컬럼의 카드를 마우스로 드래그한다.
2. In-Progress 컬럼 위에서 드롭한다.
3. **결과**: 카드가 이동하고, 다른 기기에서 새로고침해도 동일 위치에 표시된다.

---

### 시나리오 5 — 작업 완료 처리
> 카드를 Done 컬럼으로 이동한다.

1. In-Progress 컬럼의 카드를 Done 컬럼으로 드래그 앤 드롭한다.
2. **결과**: 카드가 Done 컬럼으로 이동하고 완료 상태로 표시된다.

---

### 시나리오 6 — 불필요한 카드 삭제
> 취소된 작업의 카드를 제거한다.

1. 삭제할 카드에 마우스를 올려 × 버튼을 클릭한다.
2. **결과**: 카드가 DB에서 삭제되고 보드에서 사라진다.

---

### 시나리오 7 — 다중 기기 접근
> 다른 기기에서 같은 계정으로 로그인한다.

1. 스마트폰 브라우저에서 앱에 접속해 로그인한다.
2. **결과**: PC에서 만든 카드가 동일하게 표시된다.

## 7. 성공 기준

### 기능적 성공 기준 (Functional Acceptance Criteria)

| ID | 기준 | 검증 방법 |
|----|------|----------|
| AC-01 | 로그인 전에는 로그인 화면만 표시된다 | 비로그인 상태에서 접속 확인 |
| AC-02 | 이메일/비밀번호로 회원가입 및 로그인이 된다 | 신규 계정 생성 후 로그인 |
| AC-03 | Google / GitHub 소셜 로그인이 된다 | 각 소셜 버튼 클릭 후 보드 진입 확인 |
| AC-04 | 로그아웃 버튼 클릭 시 로그인 화면으로 복귀한다 | 로그아웃 후 화면 전환 확인 |
| AC-05 | 카드 추가/이동/삭제가 Supabase DB에 반영된다 | Dashboard Table Editor에서 데이터 확인 |
| AC-06 | 다른 계정의 카드는 보이지 않는다 | 두 계정으로 각각 로그인하여 데이터 격리 확인 |
| AC-07 | 드래그 앤 드롭으로 카드가 컬럼 간 이동된다 | 모든 컬럼 조합 이동 테스트 |
| AC-08 | 빈 입력창으로 카드 추가 시 무시된다 | 공백 입력 후 추가 버튼 클릭 |

### 비기능적 성공 기준 (Non-functional Acceptance Criteria)

| ID | 기준 | 측정 방법 |
|----|------|----------|
| NF-01 | 초기 로드 2초 이내 | 브라우저 DevTools Network 탭 |
| NF-02 | Chrome, Firefox, Edge 최신 버전에서 모든 기능 동작 | 각 브라우저에서 시나리오 수행 |
| NF-03 | RLS로 사용자 데이터가 격리된다 | Supabase Dashboard에서 RLS 정책 확인 |
