## 2. 목적

개인 혹은 소규모 팀이 작업 항목을 시각적으로 관리할 수 있는 칸반보드 웹 애플리케이션을 제공한다. 사용자는 카드를 생성·이동·삭제하며 TO-DO → In-Progress → Done 흐름으로 작업 상태를 추적할 수 있다.

---

## 3. 목표 사용자

- 개인 태스크를 체계적으로 관리하고 싶은 개발자 및 기획자
- 별도의 설치 없이 브라우저만으로 사용할 수 있는 경량 도구가 필요한 사람

---

## 4. 핵심 기능 (MVP)

| # | 기능 | 설명 |
|---|------|------|
| F-01 | 컬럼 구성 | TO-DO, In-Progress, Done 3개 컬럼 고정 표시 |
| F-02 | 카드 추가 | 각 컬럼 하단 입력창에 텍스트 입력 후 Enter 또는 버튼 클릭으로 카드 생성 |
| F-03 | 카드 삭제 | 카드의 × 버튼 클릭으로 카드 제거 |
| F-04 | 드래그 앤 드롭 | 카드를 다른 컬럼으로 드래그하여 상태 이동 (HTML5 Drag and Drop API) |
| F-05 | 상태 영속성 | localStorage에 상태 저장, 새로고침 후에도 데이터 유지 |

## 6. 사용 시나리오

### 시나리오 1 — 새 작업 등록
> 사용자가 오늘 해야 할 일을 보드에 기록한다.

1. 브라우저에서 `index.html`을 연다.
2. TO-DO 컬럼 하단 입력창에 "디자인 시안 검토"를 입력한다.
3. Enter 키를 누른다.
4. **결과**: TO-DO 컬럼에 "디자인 시안 검토" 카드가 생성된다.

---

### 시나리오 2 — 작업 진행 상태 변경
> 작업을 시작하면서 카드를 In-Progress 컬럼으로 이동한다.

1. TO-DO 컬럼의 "디자인 시안 검토" 카드를 마우스로 드래그한다.
2. In-Progress 컬럼 위에서 드롭한다.
3. **결과**: 카드가 In-Progress 컬럼으로 이동하고, 페이지를 새로고침해도 위치가 유지된다.

---

### 시나리오 3 — 작업 완료 처리
> 작업이 완료되면 카드를 Done 컬럼으로 이동한다.

1. In-Progress 컬럼의 카드를 Done 컬럼으로 드래그 앤 드롭한다.
2. **결과**: 카드가 Done 컬럼으로 이동하고 완료 상태로 표시된다.

---

### 시나리오 4 — 불필요한 카드 삭제
> 취소된 작업의 카드를 제거한다.

1. 삭제할 카드에 마우스를 올려 × 버튼을 표시한다.
2. × 버튼을 클릭한다.
3. **결과**: 카드가 보드에서 제거되고, 새로고침 후에도 복구되지 않는다.

---

### 시나리오 5 — 브라우저 재방문 시 상태 복원
> 브라우저를 닫았다가 다시 열어도 작업 목록이 그대로 남아 있다.

1. 보드에 카드를 여러 개 추가하고 컬럼 간 이동도 수행한다.
2. 브라우저 탭을 닫는다.
3. 다시 `index.html`을 연다.
4. **결과**: 마지막 상태 그대로 모든 카드와 컬럼 위치가 복원된다.

## 7. 성공 기준

### 기능적 성공 기준 (Functional Acceptance Criteria)

| ID | 기준 | 검증 방법 |
|----|------|----------|
| AC-01 | TO-DO, In-Progress, Done 컬럼이 항상 표시된다 | 브라우저에서 화면 확인 |
| AC-02 | 입력창에 텍스트를 입력하고 Enter 또는 버튼 클릭 시 카드가 해당 컬럼에 생성된다 | 각 컬럼에서 카드 추가 시도 |
| AC-03 | 빈 입력창으로 추가 시도 시 카드가 생성되지 않는다 | 공백 입력 후 추가 버튼 클릭 |
| AC-04 | 카드를 다른 컬럼으로 드래그 앤 드롭하면 해당 컬럼으로 이동한다 | 세 컬럼 간 모든 조합 이동 테스트 |
| AC-05 | 같은 컬럼 내 드롭 시 카드 위치가 변경되지 않는다 | 같은 컬럼 내 드롭 후 상태 확인 |
| AC-06 | × 버튼 클릭 시 카드가 삭제된다 | 카드 삭제 후 목록 확인 |
| AC-07 | 페이지 새로고침 후 카드 목록과 컬럼 위치가 그대로 유지된다 | 카드 추가·이동 후 F5 새로고침 |
| AC-08 | 드래그 중 컬럼에 시각적 하이라이트가 표시된다 | 드래그 중 컬럼 배경색 변화 확인 |

### 비기능적 성공 기준 (Non-functional Acceptance Criteria)

| ID | 기준 | 측정 방법 |
|----|------|----------|
| NF-01 | 정적 파일 초기 로드 1초 이내 | 브라우저 DevTools Network 탭 |
| NF-02 | 외부 라이브러리 의존성 없음 | Network 탭에서 외부 CDN 요청 없음 확인 |
| NF-03 | Chrome, Firefox, Edge 최신 버전에서 모든 기능 동작 | 각 브라우저에서 시나리오 1~5 수행 |
| NF-04 | localStorage 데이터가 올바른 JSON 형식으로 저장된다 | DevTools Application 탭에서 `kanban-cards` 키 확인 |
