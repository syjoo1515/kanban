# User Flow — Kanban Board

## 1. 전체 흐름 개요

```
앱 진입
  └─ localStorage에 저장된 데이터 로드
       ├─ 데이터 있음 → 카드 렌더링
       └─ 데이터 없음 → 빈 보드 표시
            ↓
       [보드 화면]
       ├─ 카드 추가
       ├─ 카드 삭제
       └─ 카드 이동 (드래그 앤 드롭)
```

---

## 2. 카드 추가 흐름

```
사용자가 컬럼 하단 입력창 클릭
  └─ 텍스트 입력
       ├─ Enter 키 입력 또는 "추가" 버튼 클릭
       │    ├─ 텍스트가 비어있음 → 아무 동작 없음
       │    └─ 텍스트 있음
       │         ├─ 새 카드 객체 생성 { id, text, column }
       │         ├─ 상태 배열에 추가
       │         ├─ localStorage 저장
       │         ├─ 보드 재렌더링
       │         └─ 입력창 초기화
       └─ ESC 키 → 입력 취소 (입력창 포커스 해제)
```

---

## 3. 카드 삭제 흐름

```
사용자가 카드의 × 버튼 클릭
  └─ 해당 카드 id로 상태 배열에서 제거
       ├─ localStorage 저장
       └─ 보드 재렌더링
```

---

## 4. 카드 이동 (드래그 앤 드롭) 흐름

```
사용자가 카드를 드래그 시작 (dragstart)
  └─ 카드 id를 dataTransfer에 저장
       ↓
  드래그 중 컬럼 위 통과 (dragover)
  └─ 드롭 허용 (preventDefault)
       ↓
  컬럼 진입 (dragenter)
  └─ 컬럼 하이라이트 활성화
       ↓
  컬럼 이탈 (dragleave)
  └─ 컬럼 하이라이트 비활성화
       ↓
  컬럼에 드롭 (drop)
  └─ dataTransfer에서 카드 id 읽기
       ├─ 같은 컬럼에 드롭 → 변경 없음
       └─ 다른 컬럼에 드롭
            ├─ 카드의 column 값 변경
            ├─ localStorage 저장
            └─ 보드 재렌더링
```

---

## 5. 화면 상태 다이어그램

```
[빈 보드]
    │
    │ 카드 추가
    ▼
[카드 있는 보드]
    │           │           │
    │ 드래그     │ 삭제       │ 추가
    ▼           ▼           ▼
[카드 이동]  [카드 제거]  [카드 추가]
    │           │           │
    └───────────┴───────────┘
                │
         localStorage 동기화
                │
         보드 재렌더링
```

---

## 6. 엣지 케이스

| 상황 | 처리 방법 |
|------|----------|
| 입력창이 비어 있을 때 추가 | 아무 동작 없음 (공백만 있을 경우 trim 후 무시) |
| 같은 컬럼 내 드롭 | 상태 변경 없이 무시 |
| localStorage 없는 환경 | 빈 배열로 초기화, 저장 시도 시 에러 무시 |
